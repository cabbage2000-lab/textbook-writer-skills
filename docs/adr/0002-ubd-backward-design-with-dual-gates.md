# ADR 0002:以 UbD 逆向设计为内核,核心 gate 前移到"预期结果"

## 状态

已接受。

## 背景

调研报告(本地文档,不纳入版本库)的核心结论:现有开源方案的工程结构已经成熟(tutorial-writer 的五阶段流水线、强制 gate、交接契约可直接借鉴),但没有任何 skill 把"怎么把一门知识教明白"的教学法内核装进去。直接生成的"教材"有三个通病:知识点堆砌无主线、例题习题无认知梯度、评估与内容脱节。

tutorial-writer 的唯一强制停点设在"大纲"阶段——对教程够用,因为教程的目标天然清晰(教会一件具体的事)。教材不同:如果作者没先想清楚"学生学完该带走什么",大纲本身就已经是无主线堆砌,**在大纲处才 gate 为时已晚**。

## 决策

- 以 UbD(Understanding by Design)逆向设计三阶段为内核组织五阶段流水线:教学定位确认 → 预期结果设计(UbD 阶段一)→ 评估与章节设计(UbD 阶段二+三前奏)→ 章节正文编写(UbD 阶段三)→ 审核定稿;
- **核心 gate 设在阶段 2**:UbD 五件套(大概念 / 持久理解 / 核心问题 / 迁移目标 / 学习目标)必须完整呈现给作者,经明确确认才放行;
- **次要 gate 设在阶段 3**:章节树(每章强制对齐持久理解,未对齐的章要砍或改)+ Bloom 梯度报告(单层堆积 ≥60% 或缺应用/分析层必须告警),经作者确认才进入写作;
- gate 判定规则:"明确确认"指明确表示通过且不附带任何修改要求;"确认,但把 X 改一下"按修改分支处理,修改优先于确认;修改后重新完整呈现,循环直到明确确认;
- Bloom 认知层级作为统一标尺,贯穿学习目标动词、例题习题标注与全书梯度检查。

## 后果

流程比 tutorial-writer 重:作者必须先回答"学生该带走什么"才能见到大纲,这一轮思考不可跳过。对策是在 `references/ubd-framework.md` 和 `bloom-levels.md` 里给足示例降低思考成本,而不是弱化 gate。

双 gate 是 PRD 红线:任何"优化流程"的改动都不得绕过或自动通过 gate。若未来发现 gate 处返工率高,改进方向是提升五件套与章节树草案的首轮质量,而不是取消停点——停点本身就是本 skill 区别于"长文生成器"的产品内核。
