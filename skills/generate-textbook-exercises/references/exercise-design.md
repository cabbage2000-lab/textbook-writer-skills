# STEM 例题/习题设计与验证规范

本文档是 generate-textbook-exercises skill 的出题唯一权威来源，也被 chapter-template.md 的例题段落引用。核心使命：**每一个进入教材的答案都必须是真算过的**。

## 1. 三类题的定位差异

| 类型 | 定位 | 支撑程度 | 数量/章 |
| ------ | ------ | ---------- | --------- |
| 示范例题 | 教——完整解题过程给读者看 | 全脚手架（每步带理由） | 1–3 |
| 引导练习 | 扶——读者半独立地做 | 部分脚手架（提示阶梯） | 1–3 |
| 独立习题 | 放——读者独立完成 | 无脚手架（仅附参考答案） | 3–8 |

**与 Bloom 梯度的正交关系**：三类题是"支撑程度"维度，Bloom 是"认知高度"维度，**不可混为一谈**。示范例题可以是分析层（示范一个错误诊断），独立习题也可以是理解层（独立解释一个概念）。不存在"引导练习=理解层"之类的绑定。层级判定标准见 [../../design-textbook-outline/references/bloom-levels.md](../../design-textbook-outline/references/bloom-levels.md) 第 4 节。

## 2. STEM 题型分类与验证方式

| 题型 | 示例 | 验证方式 |
| ------ | ------ | ---------- |
| 数值计算 | 解方程组、求行列式、数列求和 | 用 Python（sympy/numpy）独立复算最终答案；环境无 Python 时手算两遍交叉核对 |
| 符号推导 | 化简、求导、证明恒等式 | 逐步核验每步依据 + sympy 符号验证（可行时） |
| 证明题 | 证明性质/定理 | 核验证明链条每步依据（引用的定义/定理是否已学）；无法机器验证时标注"需作者确认" |
| 代码题 | 实现算法、复现数值实验 | 实际运行代码核对输出 |
| 概念辨析 | 判断正误 + 说明理由 | 核对定义来源（教材设计文档或学科公认定义） |
| 开放/探究题 | 数值实验设计、建模讨论 | 不设标准答案，给"参考要点"，标注"开放题" |

## 3. 答案验证流程（红线，逐条硬性）

1. **出题时同步写完整解答**，禁止先攒题目后补答案的分离式编造；
2. **计算类必须独立复算**：用与解答不同的路径复算（优先写一段 Python 用 Bash 实际执行；解答用行变换求的，验证可用 numpy 直接求），比对最终结果；
3. **复算不一致 → 修题或修解答后重新验证；连续 2 次验证失败 → 弃题换题**，不允许"大概率对"的答案进入教材；
4. **验证状态标注只有两种**：`✅ 已验证（<方式>）` / `⚠️ 需作者确认（<原因>）`，不存在第三种含糊状态；
5. **独立习题的参考答案同样走验证**；给"解题路径"的题至少含关键中间结果（可核对的锚点），不能只写"同例 N-M 方法";
6. **"留给读者作为练习"式悬空 = 违规**。任何题都必须有参考答案、解题路径或参考要点三者之一。

验证用的复算代码或复算过程**随题附上**（放在"验证"行之后的代码块中），供作者复查。

## 4. 三类题各自的设计要点

### 示范例题

- 题面紧贴本章刚讲的概念，不引入未讲知识；
- 解答**每步一行理由**——"两边同乘 A 的逆（A 可逆由上一步行列式非零保证）"，而不是裸算式；
- 末尾加"回顾：这道题用到了<概念>的<哪个性质>"，把题拉回概念主线。

### 引导练习

- 与某道示范例题**同构但换数据/换场景**——练的是同一认知程序；
- 提示阶梯固定两层：提示 1（方向性，指出从哪入手）→ 提示 2（关键步骤，给中间关键变形）→ 完整解答（供对照）；
- 提示不能直接泄露答案，完整解答同样走验证流程。

### 独立习题

- 三种构成：**变式题**（示范例题换条件）、**组合题**（跨概念综合）、**开放题**（少量，可选）；
- 按 Bloom 阶梯先低后高排列；
- 开放题给参考要点，不编造标准答案。

## 5. Bloom 标注规范

- 每题必标，格式 `【Bloom：应用】`，紧跟题目标题；
- 层级判定按 bloom-levels.md 第 4 节（看认知操作不看表面难度）；
- 同题多问取**最高**层级；判定歧义时**就低不就高**；
- 标注结果按输出契约回传调用方（供全书梯度统计），不许漏标。

## 6. 输出格式

正文格式（与 chapter-template.md 第 4 节一致，全书统一）：

```markdown
#### 例 3-1：矩阵乘法的行视角【Bloom：理解】

**题目**：……

**解**：（逐步推导，每步一行理由）

**验证**：✅ 已验证（Python/sympy 复算，结果一致）

#### 习题 3-2【Bloom：应用】

**题目**：……

**参考答案**：……（或 **解题路径**：关键步骤 1 → 2 → 3）
```

编号规则：`例 <章号>-<序号>`、`习题 <章号>-<序号>`，章内各自从 1 连续编号；引导练习计入"例"编号序列，标题后缀"（引导练习）"。类型与正文标记的对应关系（供对账）：`例` 前缀且无后缀 = 示范例题；`例` 前缀 +"（引导练习）"后缀 = 引导练习；`习题` 前缀 = 独立习题。

对调用方（write-textbook-chapter）的输出契约：`{题目[]}`，每题 `{编号, 类型, 题干, 解答或提示, Bloom层级, 验证状态}`（类型 ∈ 示范例题/引导练习/独立习题）。

## 7. 反模式清单（出现即返工）

1. **全章题目同一 Bloom 层级**——没有认知阶梯，违背梯度设计初衷；
2. **题目与本章学习目标无关**——好题但不服务本章，删或移；
3. **解答跳步**——"易得""显然可知"是教材大忌，每步都要有着落；
4. **无验证直接给答案**——红线违规，任何理由都不接受；
5. **文科论述题混入**——v1 只做 STEM 概念型；遇到论述/案例分析类需求，标注"（v2 支持）"并改出概念辨析题替代；
6. **提示泄底**——引导练习的提示 1 直接给出完整做法，等于没有阶梯；
7. **开放题假装有标准答案**——开放题必须明示"开放题"并只给参考要点。
