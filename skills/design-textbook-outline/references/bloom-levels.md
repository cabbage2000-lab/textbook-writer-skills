# Bloom 认知层级（修订版）：学习目标与例题梯度的标尺

本文档是全项目 Bloom 层级的唯一权威来源：design-textbook-outline 用它写学习目标和梯度报告，generate-textbook-exercises 用它判定题目层级，write-textbook 主 skill 在阶段 5 用它复核实际梯度。

## 1. 六层级总览

修订版 Bloom 分类法六个认知层级，由低到高（首次出现注英文，其后全书只用中文）：

| 层级 | 一句话判据 | 教材场景典型题型 |
| ------ | ----------- | ----------------- |
| 记忆（remembering） | 能原样取出学过的信息 | 写出定义、列举性质、指出符号含义 |
| 理解（understanding） | 能用自己的方式重新表达信息 | 解释为什么、举一个新例子、判断正误并说明理由 |
| 应用（applying） | 能把学过的程序用到新数据上 | 计算、求解、按算法执行一遍 |
| 分析（analyzing） | 能拆解结构、看出成分之间的关系 | 推导、诊断错误解法、比较两种方法的适用场景 |
| 评价（evaluating） | 能依据标准做出判断并辩护 | 论证某方法更优、评估一个证明是否严密 |
| 创造（creating） | 能组合出新的结构或方案 | 设计算法、证明新命题、把方法推广到新对象 |

## 2. 中文动词表（写学习目标用）

每条学习目标必须从对应层级选动词，禁用"理解/掌握/了解/熟悉"等不可验证动词。

| 层级 | 动词 |
| ------ | ------ |
| 记忆 | 列举、复述、识别、写出定义、指出、命名、默写 |
| 理解 | 解释、概括、举例说明、用自己的话描述、判断正误并说明理由、翻译（在不同表示形式间转换）、归类 |
| 应用 | 计算、求解、运用、演示、将某方法套用到新数据、按步骤执行、验证（代入检验） |
| 分析 | 推导、分解、区分、找出关系、诊断错误、比较适用场景、归因、化归 |
| 评价 | 论证、评估优劣、选择并辩护、批判、检验严密性、权衡 |
| 创造 | 设计、构造、提出新解法、组合、证明新命题、推广、建模 |

## 3. 可验证学习目标的写法

**公式**：条件 + Bloom 动词 + 对象 + 可观察标准。

三组"模糊 → 可验证"改写示例：

1. ❌ "理解矩阵乘法" → ✅ "能计算 2×3 与 3×2 矩阵的乘积，并解释为何交换相乘后结果维度不同（Bloom：应用）"
2. ❌ "掌握高斯消元法" → ✅ "给定 3×4 增广矩阵，能用高斯消元法化为行阶梯形并判断解的个数（Bloom：应用）"
3. ❌ "了解特征值的意义" → ✅ "能用自己的话解释特征向量在变换下'方向不变'的几何含义，并举出一个非教材例子（Bloom：理解）"

写完后的检验法：能否据此直接出一道题？出不了题的目标就是不可验证的，重写。

## 4. 例题/习题的层级判定

判定一道题属于哪一层，看**学生解题时实际调用的认知操作**，不看题目表面难度。同一知识点在六个层级各长什么样（以"等差数列求和"为例；下表是判定标尺的示意，不宜原样搬进教材当正式题目——至少换数据、换场景）：

| 层级 | 题目示例 |
| ------ | --------- |
| 记忆 | 写出等差数列前 n 项和公式。 |
| 理解 | 解释为什么求和公式里有 (首项 + 末项)：它对应哪个配对操作？ |
| 应用 | 求 3, 7, 11, …, 399 的和。 |
| 分析 | 某同学计算 3+7+11+…+399 时，先求项数 n = (399−3)/4 = 99，再代入求和公式得 S = 99×(3+399)/2 = 19899。找出错误所在并改正。 |
| 评价 | 比较"配对法"和"倒序相加法"两种推导，哪种更容易推广到等比数列？说明理由。 |
| 创造 | 构造一个现实场景问题，其解需要用到等差数列求和，并给出完整解答。 |

**判定原则**：

- 同一道题多问时，取**最高**层级标注；
- 判定歧义时**就低不就高**（宁可把一道"疑似分析"的题标成应用，也不虚标高层级——虚标会让梯度报告失真）。

## 5. 梯度检查规则（全书级）

**统计口径**：全书所有题目按 Bloom 层级计数，示范例题、引导练习、独立习题都计入。

- **规则 1（单层堆积）**：任一层级占比 ≥60% → 告警：`⚠️ 梯度告警：<层级> 层占比 N%，题目过度堆积`，并列出建议补题的章。
- **规则 2（缺高层）**：应用 + 分析两层合计 = 0 题 → 告警：`⚠️ 梯度告警：全书没有"应用/分析"层题目，缺认知阶梯上段`。
- **规则 3（缺低层入门，提示级）**：记忆 + 理解两层合计 = 0 题 → 提示：`ℹ️ 全书没有低层级入门题，确认读者起点是否足够高`。

规则 1、2 是告警级：必须连同补题建议一起呈现给作者，作者确认后才放行（阶段 3 gate 的组成部分）。规则 3 是提示级：呈现即可，不阻塞。

## 6. 梯度报告格式

固定为"层级 × 章节计数矩阵 + 合计行 + 占比行 + 告警/提示块"。示例（假数据）：

```markdown
### 全书 Bloom 梯度报告

| 章节 | 记忆 | 理解 | 应用 | 分析 | 评价 | 创造 | 小计 |
| ------ | ------ | ------ | ------ | ------ | ------ | ------ | ------ |
| 01 向量与线性组合 | 1 | 3 | 2 | 0 | 0 | 0 | 6 |
| 02 矩阵与线性变换 | 1 | 4 | 2 | 1 | 0 | 0 | 8 |
| 03 解线性方程组 | 0 | 2 | 4 | 1 | 0 | 0 | 7 |
| **合计** | 2 | 9 | 8 | 2 | 0 | 0 | 21 |
| **占比** | 10% | 43% | 38% | 10% | 0% | 0% | 100% |

**检查结果**：
- 规则 1：最高占比"理解"43% < 60%，通过。
- 规则 2：应用 + 分析 = 10 题 > 0，通过。
- 规则 3：记忆 + 理解 = 11 题 > 0，通过。
- ℹ️ 备注：评价/创造两层全书为 0，属常见情况；建议在末章表现性任务附近补 1–2 道（非强制）。
```

若有告警，在检查结果块中按规则文案输出，并逐条给出"建议在第 N 章补 X 层级题目"的可操作建议。

## 7. 每章内部的梯度建议（非硬规则）

三类题（示范例题/引导练习/独立习题）是**支撑程度**维度，Bloom 是**认知高度**维度，二者正交、不可互相绑定。但对多数章节，以下经验分布是好的默认：

- 示范例题：落在记忆～应用层（教新东西时先低后中）；
- 引导练习：主落应用层（半独立地练刚学的程序）；
- 独立习题：应用～分析为主，按 Bloom 阶梯**先低后高**排列；
- 评价/创造：全书少量点缀即可（1–3 题，常放在末章或表现性任务中），不必每章都有。

偏离此分布不是错误——例如"证明技巧"主题章可能整体偏分析层。梯度以**全书**为单位检查（第 5 节规则），章内分布只是设计参考。
